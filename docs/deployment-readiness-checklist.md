# Deployment Readiness Checklist

## Workflow identity
- Workflow name is clear
- Owner is assigned
- Purpose is documented

## Inputs and outputs
- Inputs are defined
- Outputs are defined
- Required fields are documented

## Failure handling
- Retry policy exists
- Fallback path exists
- Manual recovery process exists

## Logging and alerting
- Structured logs are enabled
- Failure alerts are routed correctly
- Run IDs are captured

## Safety checks
- Duplicate prevention is in place
- Secrets are stored securely
- Sensitive data logging is minimized

## Testing
- Happy path tested
- Failure paths tested
- Edge cases tested

## Final signoff
- Documentation complete
- Known risks documented
- Deployment approved
